### Why the Pattern Approach Works

Instead of solving 300 random problems and hoping the right one shows up, solve 10–15 problems per pattern and learn to recognize the template on sight. The same sliding window logic that solves "maximum sum subarray of size K" also solves "longest substring with K distinct characters."

## Merged from DSA-Roadmap.md

### Recommended Study Order

**Foundation → Complexity → Problem Solving → Patterns → Speed**

1. Big O → Arrays → Strings → Recursion → Backtracking
2. Two Pointers → Sliding Window → Binary Search → Prefix Sum
3. Stack → Queue → Linked List → Trees → BST → Heaps
4. Graphs → BFS → DFS → Shortest Path → Topological Sort
5. Greedy → Divide & Conquer → Dynamic Programming

**Practice progression:** Easy → Medium → Hard → Repeat → Master

**Per-problem approach:** Read Problem → Think → Brute Force → Optimize → Code
