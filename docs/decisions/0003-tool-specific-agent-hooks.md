# ADR-0003: Tool-Specific Agent Hook Files

## Context

Sensei instances are operated by LLM coding agents (Claude Code, Cursor, Kiro, Copilot, Windsurf, Cline, Roo Code, JetBrains AI, Aider, and others). The boot chain established in [ADR-0002](0002-agent-bootstrap.md) requires the agent to read `AGENTS.md` at the instance root.

In practice, not all tools auto-discover `AGENTS.md`. Claude Code reads `CLAUDE.md`. Kiro uses `.kiro/steering/`. Cursor uses `.cursor/rules/`. GitHub Copilot uses `.github/copilot-instructions.md`. A fresh session in a tool that doesn't auto-read `AGENTS.md` produces a broken boot chain — the agent doesn't recognize Sensei conventions.

The AGENTS.md open spec (Linux Foundation Agentic AI Foundation, Dec 2025) is gaining adoption — Codex, Copilot, Cursor, Windsurf, Cline, Roo Code, and others now read it natively. But Claude Code and tool-specific config directories remain parallel paths that need bridging.

The analyst review explicitly called this out as the largest blocker in Sensei's "agent-agnostic" claim: without shims, "works with any LLM agent" is aspirational marketing.

## Decision

`sensei init` generates thin shim files for all major LLM coding tools alongside `AGENTS.md`. Each shim points to `AGENTS.md` as the single source of truth — no content is duplicated.

Files generated by `sensei init`:

| File | Tool | Content |
|---|---|---|
| `AGENTS.md` | Universal (Codex, Copilot, Cursor, Windsurf, Cline, Roo Code, Devin, Amp, and any tool adopting the AGENTS.md spec) | Full boot document |
| `CLAUDE.md` | Claude Code | `See @AGENTS.md` (native import directive) |
| `.kiro/steering/sensei.md` | Kiro CLI | Prose pointer to `AGENTS.md` |
| `.cursor/rules/sensei.mdc` | Cursor | MDC frontmatter + prose pointer |
| `.github/copilot-instructions.md` | GitHub Copilot | Prose pointer |
| `.windsurf/rules/sensei.md` | Windsurf | YAML frontmatter + prose pointer |
| `.clinerules/sensei.md` | Cline | Prose pointer |
| `.roo/rules/sensei.md` | Roo Code | Prose pointer |
| `.aiassistant/rules/sensei.md` | JetBrains AI | Prose pointer |

All shims are generated by default. No flag required. The cost of unused shim files is near zero (1–3 lines each); the cost of a broken boot chain is a confused user on first session.

Tool-specific directories (`.kiro/`, `.cursor/`, `.github/`, etc.) are committed to version control so collaborators get the same boot experience.

## Alternatives Considered

- **Generate only `AGENTS.md`.** Rejected because Claude Code doesn't read it natively. A fresh Claude Code session in a Sensei instance would fail to boot, directly breaking the "any LLM agent" promise.
- **Duplicate `AGENTS.md` content into each tool file.** Rejected because it creates N copies that drift. `AGENTS.md` is the single source of truth; shims point to it.
- **Flag-gated generation (`--hooks kiro,cursor`).** Rejected because it requires the user to know which tool they'll use at init time. Generating all is simpler and the files are tiny.
- **Auto-detect tool from environment.** Rejected because a Sensei home may be opened by different agents across different sessions (one at home in Claude Code, another on a laptop with Cursor). Generating all covers everyone.

## Consequences

Every major LLM coding tool discovers Sensei's boot chain on first session without user intervention. `AGENTS.md` remains the canonical boot document. Tool-specific files are 1–3 line shims that never need updating unless the tool's convention changes. The `.gitignore` template must not ignore tool directories.

Claude Code's `@AGENTS.md` import is the only true file-inclusion mechanism; all other shims use prose instructions ("Read and follow AGENTS.md") which work because these are agentic tools with file-reading capabilities.

Future tools that adopt the AGENTS.md spec need no shim — they work automatically.
